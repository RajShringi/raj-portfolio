Recently, I was building an expense tracker app. I was working on search functionality in the transactions table. The user
can search for transactions by category name or transaction note.

Here is the bird's-eye view of the functionality.

- There is a search input where the user types the search query.
- Typing the search query triggers a backend request.
- I get the result based on the search query and update the table.

```js
"use client";
import { Input } from "@/components/ui/input";
import { ChangeEvent, useEffect, useState } from "react";
import { useRouter, useSearchParams } from "next/navigation";

export default function TransactionSearchInput() {
  const searchParams = useSearchParams();
  const router = useRouter();

  const [search, setSearch] = useState<string>(
    searchParams.get("search") ?? "",
  );

  const handleSearch = (e: ChangeEvent<HTMLInputElement>) => {
    setSearch(e.target.value);
  };

  function updateSearchParams() {
    const params = new URLSearchParams(searchParams);
    if (search) {
      params.set("search", search);
    } else {
      params.delete("search");
    }
    params.set("page", "1");
    router.replace(`?${params.toString()}`);
  }

  useEffect(() => {
    updateSearchParams();
  }, [search]);

  return (
    <Input
      type="text"
      name="search"
      value={search}
      onChange={handleSearch}
      placeholder="Search transactions by note and category..."
      className="bg-white border border-lime-200 focus-visible:ring-lime-200"
    />
  );
}
```

So now you may think functionality is there, everything is working, so what's the problem? As we can see, we are sending a
request to the server on every keystroke. So why is this bad?

```text
User types: bills

b     -> request
bi    -> request
bil   -> request
bill  -> request
bills -> request
```

1. For every keystroke, we are processing a backend request. Suppose the user types `bills`. Now we are getting 5 requests,
   instead of 1. On every keystroke, our state changes, which re-render our component, useEffect runs, then request goes to the server,
   our database gets queried and we get response and then our UI updates. This whole process happens on every keystroke. We are making
   unnecessary api
   calls.
2. These unnecessary api calls are increasing backend load. Every time the server receives the request, the database gets
   queried, and scalability becomes worse.
3. This also affects user experience. UI feels less smooth because of unnecessary rendering and updating.

## So What's The Solution?

Here, debounce comes into the picture. Debounce is a technique to delay the execution of a function until a specified amount
of time has passed since the last user action. Instead of sending a request immediately, we can wait until the user pauses typing and
then send a single request with the final value. Here is the debounced version of the code.

```js
"use client";
import { Input } from "@/components/ui/input";
import { ChangeEvent, useEffect, useState } from "react";
import { useRouter, useSearchParams } from "next/navigation";

export default function TransactionSearchInput() {
  const searchParams = useSearchParams();
  const router = useRouter();

  const [search, setSearch] = useState<string>(
    searchParams.get("search") ?? "",
  );

  const handleSearch = (e: ChangeEvent<HTMLInputElement>) => {
    setSearch(e.target.value);
  };

  function updateSearchParams() {
    const params = new URLSearchParams(searchParams);
    if (search) {
      params.set("search", search);
    } else {
      params.delete("search");
    }
    params.set("page", "1");
    router.replace(`?${params.toString()}`);
  }

  useEffect(() => {
    const timeout = setTimeout(() => {
      updateSearchParams();
    }, 500);

    return () => {
      clearTimeout(timeout);
    };
  }, [search]);

  return (
    <Input
      type="text"
      name="search"
      value={search}
      onChange={handleSearch}
      placeholder="Search transactions by note and category..."
      className="bg-white border border-lime-200 focus-visible:ring-lime-200"
    />
  );
}
```

In the above code, we are delaying the function execution using setTimeout for `500 ms` until the user stops typing. This
part is a debounce.

```js
useEffect(() => {
  const timeout = setTimeout(() => {
    // api call or route update
  }, 500);

  return () => {
    clearTimeout(timeout);
  };
}, [search]);
```

Here is how debounce works.

```text
user types bills
b
bi
bil
bill
bills

(wait 500ms)

request
```

1. User types something in the search input.
2. A `500 ms` timer starts.
3. If the user types again before the 500 ms timer expires, the search state updates and the component re-renders. React then runs the
   cleanup function from the previous effect. Inside the cleanup function, we call `clearTimeout(timeout)`, which cancels the previous
   timer. After that, a new timer starts for the latest search value.
   > **NOTE:**
   > When a dependency changes, React first runs the cleanup function from the previous effect and then executes the effect again. This behavior allows us to cancel the previous timer before creating a new one
4. This process continues while the user is typing.
5. Once the user stops typing for `500 ms`, the setTimeout callback function executes, and our request goes to the server.
6. This way, we avoid sending requests on every keystroke.
